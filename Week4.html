This week we learned about positioning elements in HTML and improving usability. I had already incorporated many of these techniques into my project1 or blog, but I decided to do something fun with the media tag. I wanted to do something simple, but clear to the end-user so I made a button that turned blue if you hovered on desktop or red if you hovered on mobile. The @media tag automatically switches between the two colors at a screen width of 630px so if you resize the window to be narrow it will also turn red. Try it for yourself!


As you can see in the CSS style code below, I set the button:hover to be blue, but then using the media tag I change it to red if the screen width is less than 630px. This will be extremely helpful for resizing elements and rearranging them for mobile when making my static website as you can modify any CSS style element using this technique. Maybe you want flexbox row on desktop but then on the mobile version you want to flip to column so the elements can be easily visible and not tiny on the userâ€™s screen, well this is the perfect way to do it. This will certainly improve the usability on my future blog posts and web pages. 

    
    button:hover {
        background-color: blue;
        color: white;
    }
    
    @media screen and (max-width: 630px) {
        button:hover {
            background-color: red;
            color: white;
        }
    }

This week out of my own personal curiosity I wanted to see how hard it was to make images move around the screen. This would be practical for a banner with long text running along the top of a web page or just making a gif move across the screen. Turns out it was extremely easy. I learned that using the marquee tag that you can make images do a variety of things at a speed of your choice. Using this tag it is easy to make images continuously scroll between the left and right side of the screen or even bounce back and forth.

I did both, make the birds go from the right to the left of the screen and then wrap as well as bounce between both sides, above to demonstrate and selected a relatively high speed so that you could see the effect quickly, the default is quite slow on large screens. But this could be another use of the media tag as you could make the images smaller and move slower on mobile while making them move faster on the desktop version.
