This week we have continued our study of React JS and focused on components as well as states. We learned that states are essentially variables that can be changed by using a setter method. These methods can then be put into other methods and passed to components where they can then change the variables. This may seem a little complicated at first, but it makes sense after doing it a few times. I then went on to read about react redux also known as store. A store can hold an entire tree of state and only when given a specific action will it update. The purpose of having these specific states is so that the program operates constantly for the end user. For programs we have written so far it has not been necessary but once we incorporate our project 2 SQL database then we will need to manage timing between the client and the server with redux. We can have an example for retrieving data from the database then have another one for updating the client screen.
