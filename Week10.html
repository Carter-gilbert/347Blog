For this week’s blog I wanted to read and learn some more about react JS before we continue to dive into what we can use it for. A cool feature of react JS is that we can have components that can be easily duplicated using method calls rather than using code duplication like we did in the HTML section of this class. In these components we can also use variables so we can input different data into a similar structure, further avoiding code duplication. For example please look at the code below. 

            function Greeter(props) {
                return <h1>Welcome to CS347, {props.name}!</h1>;
              }
              
              function App() {
                return (
                    <div>
                        <Greeter name="Carter" />
                        <Greeter name="Jackie" />
                        <Greeter name="Cory" />
                    </div>
                );
              }

This code will print three headers all saying “Welcome to CS347, {props.name}! “, but rather than {props.name} this is replaced with Carter, Jackie, or Cory. Notice how it is essentially the same code, however, we are using the same method and passing it different name variables. This makes web development much easier and makes code much easier to debug.

Above you can see props and how we use props.name as the variable while we assign props.name by simply typing name=”” after calling the method name in the brackets. This is how a variable is passed to props. In this example props only stores names, but you can pass more than one variable to props when doing the method call. A practical example would be a program that needs to print a date. We could assign day, month, and year in the method call and to retrieve it in the method itself we would simply use props.day, props.month, and props.year. You can quickly see how powerful this can be and the endless uses for it.
