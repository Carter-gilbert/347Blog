Week 12 - November 19th

This week we learned how to incorporate API calls with react js. We learned how to make a store which allows us to change states and ultimately have a callback and dispatcher that will wait for the API to send the data before moving on. This removes errors caused by code using data that has not yet been received from the API. We also read about multi page apps which can be made using the react router. The example from the reading was a little confusing so I decided to find some external sources to clarify some things.

We first need to make a Router component with a switch statement inside to specify the pages we would like to switch between (the code below is a portion of the code required). As you can see each page needs its own route and path so react knows how to redirect. 

        <Switch>
            <Route exact path="/">
                    <Home />
                  </Route>
                <Route path="/about">
                    <About />
                  </Route>
        </Switch>

Now we need a way to actually navigate to each page, for a real website you would likely make it look nice, but for this proof of concept we can simple use the code below. Each method is for one page and all it does is make a heading. If you wanted more on the page you would use this method to add it. Essentially what goes into the return is the HTML (or js) you want to display on the screen. This is different than how we have given HTML paths in the past, but this enables us to have multiple dynamic pages. 

    
    function Home() {
        return (
        <div>
            <h2>Home Page</h2>
        </div>
        );
      }
          
      function About() {
        return (
        <div>
            <h2>About Page</h2>
        </div>
        );
      }
